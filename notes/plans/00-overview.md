# cmus-android — overall plan

Wrapper app for cmus using the termux terminal libs, per [spec](../spec.md).
This is the overview; each stage gets its own detailed plan in this folder
before implementation.

## Repo layout

```
cmus-android/
├── notes/                  spec, status.md, architecture.md, plans/
├── app/                    Android app module (Java)
│   └── src/main/{java,res,assets}
├── native/                 NDK-native build (no autotools/configure anywhere)
│   ├── CMakeLists.txt      top-level; adds deps + cmus core + plugins
│   ├── cmus/               cmus target defs + generated config headers
│   └── ports/              handwritten CMake + pregenerated headers for
│                           libs without upstream CMake (ncurses, libmad,
│                           libiconv)
├── third_party/            git submodules, all pinned:
│   ├── cmus                github.com/cmus/cmus @ master
│   ├── termux-app          terminal-emulator + terminal-view modules
│   ├── ncurses, libogg, libvorbis, opus, opusfile,
│   │   flac, libmad, wavpack, faad2, mp4v2, libiconv
├── patches/                per-submodule patch dirs (patches/cmus/*.patch, …)
├── patch.sh                vncpatch-style apply/regenerate script
├── settings.gradle, build.gradle, gradlew, .gitignore
```

## Key technical decisions

### Patch management (vncpatch pattern, adapted for submodules)

`patch.sh` mirrors `~/src/vncpatch/vncpatch.sh` but per-submodule:

- On first run per submodule: tag the pinned commit as `base`.
- Apply `patches/<name>/*.patch` with `git am` when HEAD == `base`.
- Regenerate with `git format-patch --output-directory patches/<name>/
  --no-stat --no-signature --numbered base` (unless `-n`).
- If a `git am` is mid-flight, print status and stop (same as vncpatch).

Submodule pointers stay at upstream commits; patches are the local delta.
Gradle's `preBuild` asserts patches are applied (fails with a hint to run
`./patch.sh`).

### Native build (NDK-native, no configure)

Single CMake tree driven by AGP `externalNativeBuild` in `app/`:

- **Deps with upstream CMake** (ogg, vorbis, opus, opusfile, flac, wavpack,
  faad2, mp4v2 [enzo1982 fork]): `add_subdirectory` with options forced to
  static libs, no programs/tests/docs.
- **ncurses**: handwritten CMake in `native/ports/ncurses/` compiling the
  ncursesw sources against pregenerated config headers (`ncurses_cfg.h`,
  generated `codes.c`/`names.c`/`lib_keyname.c` etc.). A one-time host script
  (documented in the port dir) produces these from ncurses' own generators;
  outputs are committed. Wide-char (`ncursesw`) build, static.
- **libmad**: trivial handwritten CMake (fixed source list + `FPM_64BIT`
  config defines).
- **libiconv**: handwritten CMake port (libiconv + libcharset sources with
  pregenerated config headers), static, linked into cmus core.
- **terminfo**: `xterm-256color` (+ aliases) compiled entry shipped in
  assets, extracted to `filesDir/terminfo` on first run, `TERMINFO` env set.
- **cmus core**: `add_executable` of the non-plugin sources, output named
  `libcmus.so` and placed in the CMake library output dir so AGP packages
  it; exec'd from `nativeLibraryDir` at runtime (standard trick for
  executables in apps). Config headers (`config/*.h`, VERSION, defaults)
  generated by CMake replacing what `./configure` would emit:
  - on: ncurses(w), iconv, pthread, dl, aaudio op, ip: vorbis opus wav
    wavpack flac mad aac (faad2) mp4/m4a (mp4v2+faad2), android IPC
  - off: cddb, cdio, ffmpeg, alsa/pulse/jack/oss/…, samplerate, rtsched
- **plugins**: each ip/op plugin is a normal `SHARED` lib
  (`libcmus_ip_flac.so`, `libcmus_op_aaudio.so`, …), deps linked statically
  into each plugin.
- **ABI**: arm64-v8a only (min Android 14; real-device testing).
- The upstream aaudio output plugin (op/aaudio.c, already in cmus master)
  is used as-is.

### Runtime layout (Java side, at first run / version change)

- `filesDir/cmus-lib/{ip,op}/<name>.so` → symlinks to
  `nativeLibraryDir/libcmus_{ip,op}_<name>.so`; `CMUS_LIB_DIR` points here.
  (cmus scans `$CMUS_LIB_DIR/{ip,op}/*.so`; no cmus patch needed.)
- `filesDir/cmus-data/` ← assets copy of cmus `data/` (rc, *.theme);
  `CMUS_DATA_DIR`.
- `filesDir/cmus-home/` ← `CMUS_HOME` (config, cache, playlists, autosave).
- Spawn via termux terminal-emulator JNI (`forkpty`), env: HOME, TMPDIR,
  TERM=xterm-256color, TERMINFO, CMUS_* as above.

### cmus patches (kept minimal, self-contained)

1. **`android.c` + `android.h`** — new file modeled on mpris.c (mpris.c
   itself is never touched — it's an example only, and is simply not built
   here since CONFIG_MPRIS=n); guarded by
   `CONFIG_ANDROID`. Unix socket (SOCK_STREAM, filesystem path in app dir);
   fd added to the ui_curses select loop exactly where `mpris_fd` is.
   Protocol: newline-delimited JSON both ways.
   - **out (events)**: hooked at the existing mpris call sites (playback
     status, metadata, volume) plus an options snapshot pushed on connect
     and after each executed command — this covers "settings always
     up-to-date" including colors for theme matching.
   - **in (commands)**: raw cmus command lines, executed via the same
     `run_command`/`handle_command` path server.c uses.
   - Media-control state flows over this socket to Java instead of D-Bus.
2. **hook call sites** — small hunks in ui_curses.c (and wherever else
   `mpris_*` is called) adding the parallel `android_*` calls next to the
   existing mpris ones; mpris code paths are left as-is.
3. **Remove remote streams** — drop http:// URL handling in input.c/cmus.c
   and http.c from the build (minimality, not security).
4. Anything needing a static func from cmus core gets a minimal non-static
   wrapper declared and used only from android.c (per spec).

### App architecture (Java, modern features — records, switch patterns, etc.)

- **`TermService`** (foreground service, `mediaPlayback` type): owns the
  TerminalSession (cmus process), the IPC client, and the MediaSession.
  Survives activity death; idle-quit timer (cmus stopped + app unfocused
  for X min → send `quit`, stop service).
- **`CmusIpc`**: connects to the android.c socket; exposes typed state
  (status, track, volume, options map) via listener; sends command strings.
- **`MediaControl`**: MediaSession + notification; play/pause/next/prev/
  seek from headset/system; cover art via `MediaMetadataRetriever
  .getEmbeddedPicture()` (falls back to folder art), done Java-side off the
  track path from IPC metadata.
- **`MainActivity`**: TerminalView + chrome. All chrome colors derived from
  cmus color options over IPC (window bg/fg, statusline colors → tab bar,
  bottom bar, Android status/navigation bar).
  - top compact tab bar: text-only view selector (1–7), left quick-filter
    icon (morphs bar into live-filter search box focused on library; right
    icons become ✕ to clear/exit), right sleep-timer icon (duration
    selector; icon shows minutes left) + faint round settings icon;
    long-press settings → theme selector overlay (centered, no scrim,
    scrollable, 2 columns: colorscheme | font; applies live).
  - bottom compact bar: play/pause, repeat, shuffle, seek bar, volume
    button toggling a vertical slider (visible only when softvol on),
    add-selected-to-queue, keyboard toggle.
  - extension key row when IME visible (horizontally scrolling:
    shift/ctrl/alt · space · del/esc/tab · space · arrows · space ·
    home/end/pgup/pgdn), termux-style sticky modifiers.
  - joystick dot bottom-right (faint, minecraft-style): tap = enter,
    slide up/down = arrows, slide far left = tab.
  - touch: terminal-view defaults (tap, finger-scroll as mouse events,
    pinch to zoom font); long-press = right-click, which the app
    intercepts: native confirm dialog naming the selected item, then
    `win-remove` over IPC. No text selection.
- **Settings screen** (launched from gear icon; mostly app settings + a
  small curated cmus set): per-control enable/disable toggles, aaudio op
  options, `pause_on_output_change`, softvol on/off, idle-quit minutes,
  import/export data as tar (minimal ustar impl over CMUS_HOME), load
  tracks from local folder (READ_MEDIA_AUDIO; direct paths, no SAF).
- Fonts: a few bundled monospace ttfs for the font selector.

### termux libs

`terminal-emulator` and `terminal-view` consumed as source modules from the
termux-app submodule via `settings.gradle` projectDir redirection; patched
through the same `patch.sh` mechanism only if subclassing/`TerminalViewClient`
can't cover something (expected patches: none or near-none).

## Stages

Each stage = detailed plan in `notes/plans/NN-*.md` → approval → implement →
commit(s) → status.md entry. Stages are deliberately small — one focused
session each, ending in something buildable/verifiable.

Authorship:
- Stage plans (`notes/plans/*`) are committed as Patrick, co-authored by
  Claude — he reads all plans and works on developing them.
- Implementation commits are authored as Claude
  (`Claude Fable 5 <noreply@anthropic.com>`), unless Patrick provides
  significant technical input/direction for that stage — then author as
  Patrick with Claude as co-author.

1. **Scaffolding**: repo layout, .gitignore, gradle wrapper + skeleton app
   module, notes/architecture.md + status.md started. Empty app builds.
2. **Submodules + patch.sh**: all submodules added + pinned, vncpatch-style
   script working (no patches yet), gradle preBuild check.
3. **Native deps A**: CMake tree building the upstream-CMake deps (ogg,
   vorbis, opus, opusfile, flac, wavpack, faad2) as static libs for arm64.
4. **Native deps B**: ncurses port (pregenerated headers + CMake, terminfo
   asset), libmad port, libiconv port.
5. **cmus build**: cmus core (`libcmus.so` executable trick) + ip/op plugin
   .so targets + generated config headers. Verify: cmus runs in adb shell.
6. **Terminal MVP**: termux modules wired into gradle, TerminalView +
   foreground service + spawn cmus in pty, runtime layout (symlinks,
   terminfo, data dir), IME toggle. Verify: usable TUI, flac plays.
7. **cmus IPC patch**: android.c + call-site hooks + remote-stream removal,
   as patches via patch.sh. Verify with socat/adb.
8. **Java IPC client**: CmusIpc, typed state, command sending. Verify: app
   logs live status/track/options; commands work.
9. **Media control**: MediaSession + notification + cover art + headset
   keys.
10. **Lifecycle**: idle-quit timer, service/activity edge cases, process
    restart behavior.
11. **Chrome A**: theme color extraction from IPC options + view-selector
    tab bar + Android status/nav bar coloring.
12. **Chrome B**: bottom control bar (play/pause/repeat/shuffle/seek,
    volume button + vertical slider gated on softvol, add-to-queue,
    keyboard toggle).
13. **Input A**: extension key row (scrolling, sticky modifiers).
14. **Input B**: joystick dot; long-press → native confirm → win-remove;
    gesture/zoom polish; disable text selection.
15. **Quick filter + sleep timer** (tab-bar morphing search box; duration
    selector with countdown icon).
16. **Theme/font selector overlay** + bundled fonts.
17. **Data** (renumbered from 18 and done before the settings screen,
    amended): Music-folder refresh action (READ_MEDIA_AUDIO) + the jobs
    event/poll import guard.
18. **Settings screen**: control visibility toggles, curated cmus settings
    (aaudio op options, pause_on_output_change, softvol), idle-quit
    minutes, zip import/export of CMUS_HOME (moved from the data stage,
    tar → zip).
19. **Continuous state save**: periodically save cmus state (resume
    file, autosave/library, cache) during runtime, not just at exit —
    closes the loss window of exits that skip the SIGHUP path entirely
    (force-stop's uid SIGKILL, battery death, panics; stage 10
    documented it). Likely a small timer in the android.c patch or a
    save-ish line the app triggers.
20. **ogg/opus embedded art**: the framework extractor doesn't read
    METADATA_BLOCK_PICTURE, so MediaControl falls back to folder art
    for those — a small Java-side parser (base64 FLAC PICTURE block in
    the vorbis comment header) as another step in MediaControl's art
    chain (stage-9 note).
21. **Direct touch + joystick**: direct-touch input toggle driving cmus
    mouse mode (tap = click, long-press = right-click) with a live apply
    path; an on-demand floating joystick shown when direct touch is off.
22. **termux from source**: build terminal-emulator/-view from the
    `third_party/termux-app` submodule (their Java into `:app` via
    `sourceSets`, `termux.c` via our CMake) instead of the JitPack AAR, so
    `libtermux.so` gets NDK-r28 16 KB alignment and the jitpack dependency
    is dropped. Fixes Android 16's "not 16 KB aligned" dialog.
