# Stage 5 — cmus build (core + plugins + generated config)

Goal: cmus itself, built NDK-native from the pristine submodule (no
patches this stage). Core becomes an executable packaged as
`libcmus.so`; 10 ip/op plugins become normal SHARED libs; the config
headers cmus's hand-rolled `./configure` would emit are generated by
CMake. Verify: cmus TUI runs in an interactive adb shell and plays
audio through the aaudio op.

## Facts pinned down from the tree (cmus @ d335e90)

- `./configure` emits `config.mk` + small `config/*.h` headers
  (configure:695-708), each just `#define`s for a few vars via
  `config_header` (scripts/configure.sh:137). Sources include them as
  `#include "config/curses.h"` — quoted include, so the pristine
  submodule root is searched first (file absent there), then our `-I`
  dirs: generating into `<binary-dir>/config/*.h` works without
  touching the submodule.
- Core object list = `cmus-y` (Makefile:113) + `file.o path.o prog.o
  xmalloc.o`; `mpris.o` is CONFIG_MPRIS-gated (off for us). `main()`
  is in ui_curses.c; main.c belongs to cmus-remote, which we don't
  build (IPC comes via android.c in stage 7; the server.c socket is
  still compiled in and testable with socat).
- Compile env: `-DHAVE_CONFIG` always, `-D_FILE_OFFSET_BITS=64`,
  C11 with atomics (`-std=gnu11`); `-DVERSION=...` only for
  command_mode.c input.c ui_curses.c (main.c not built);
  `-DDEFAULT_PORT=3000` for server.c. rtsched would autodetect on
  bionic — deliberately off per overview (just don't define
  REALTIME_SCHEDULING). No WORDS_BIGENDIAN.
- Core links ncursesw iconv dl m pthread, and the executable must
  export its dynamic symbols: upstream probes `DL_LIBS="-ldl
  -Wl,--export-dynamic"` (scripts/checks.sh:658) because plugins are
  linked with undefined cmus-core symbols and resolved from the
  executable at `dlopen(RTLD_NOW)` (input.c:529, output.c:111).
  CMake: `ENABLE_EXPORTS ON` on the executable.
- Plugin discovery: scans `$CMUS_LIB_DIR/{ip,op}/*.so`, plugin name =
  filename minus `.so` (input.c:519-558, misc.c:265) — the overview's
  runtime symlink naming (`ip/flac.so` → `libcmus_ip_flac.so`) works
  unpatched.
- ui_curses.c includes plain `<curses.h>` on Linux (ui_curses.c:84) —
  our ncurses port's `gen/include` works as-is.
- op/aaudio.c includes only `<aaudio/AAudio.h>` + cmus headers.
  Upstream's configure check adds `__ANDROID_UNAVAILABLE_SYMBOLS_ARE_
  WEAK__` weak-linkage games for its API-32 calls — moot at minSdk 34;
  plain `-laaudio` from the NDK.

## Layout

- `native/cmus/CMakeLists.txt` — all targets below, added from the top
  CMakeLists with `add_subdirectory(cmus)`.
- Config headers generated at CMake configure time (file(CONFIGURE) or
  configure_file) into `${CMAKE_CURRENT_BINARY_DIR}/generated/config/`;
  that dir + the submodule root go on the include path.

## Config header values (bionic, our deps)

- curses.h: HAVE_RESIZETERM=1, HAVE_USE_DEFAULT_COLORS=1 (both
  verified present in our libncursesw.a in stage 4), HAVE_ITALIC=1
  (A_ITALIC exists in the 6.4 widec build).
- iconv.h: HAVE_ICONV=1. **Include-order matters**: bionic ships its
  own limited `iconv.h` since API 28; our libiconv `gen/include` must
  come first so GNU libiconv (and its `iconv_*` → `libiconv_*` remap)
  wins over the bionic header.
- utils.h: HAVE_BYTESWAP_H=1; wcwidth.h: HAVE_WCWIDTH=1 (bionic:
  4-byte wchar_t, wcwidth exists); xmalloc.h: HAVE_STRDUP=1,
  HAVE_STRNDUP=1; debug.h: DEBUG=1 (upstream default).
- datadir.h/libdir.h: harmless placeholders — the app always sets
  CMUS_DATA_DIR/CMUS_LIB_DIR (misc.c:265-271), as will the adb-shell
  verify.
- Off-feature headers still included by sources: mpris.h
  (CONFIG_MPRIS off), cdio.h (HAVE_CDDB off), tremor.h, mpc.h, mp4.h
  (USE_MPEG4IP off → mp4v2 API), ffmpeg.h, samplerate.h, modplug.h —
  emit with the exact off-value format `config_header` produces (read
  scripts/configure.sh:137 at implementation; y/n booleans become
  `#define`/`#undef`-style output).
- VERSION string: derive from the submodule gitlink at configure time
  (`git rev-parse --short`), e.g. `v2.12.0-<sha>` matching upstream's
  git-describe fallback chain (Makefile:1-11) — stays correct across
  pin bumps with no hardcoding.

## Targets

- **Core**: `add_executable(cmus <cmus-y sources> file.c path.c prog.c
  xmalloc.c)`, `OUTPUT_NAME libcmus, SUFFIX .so`, ENABLE_EXPORTS ON,
  RUNTIME_OUTPUT_DIRECTORY = CMAKE_LIBRARY_OUTPUT_DIRECTORY so AGP
  packages it alongside real shared libs (the wireguard-android
  pattern). Links ncursesw iconv dl m only (codecs live in the
  plugins; no liblog — stderr is fine).
- **ip plugins** (`libcmus_ip_<name>.so` via OUTPUT_NAME):
  - flac ← FLAC; vorbis ← vorbisfile (pulls vorbis+ogg); opus ←
    opusfile (pulls opus+ogg); mad ← mad-static + iconv + m (sources
    ip/mad.c + ip/nomad.c); wavpack ← wavpack; aac ← faad; mp4 ←
    mp4v2 + faad; wav ← nothing; cue ← m.
  - cue is upstream-default-on and dependency-free; the overview's ip
    list omitted it — include it (cue sheet support). Flagged as the
    one scope addition.
- **op plugin**: `libcmus_op_aaudio.so` ← `-laaudio` (NDK).
- Plugins have undefined core symbols by design. If the NDK toolchain
  injects `-Wl,--no-undefined` for SHARED libs, counter per-target
  with `-Wl,-z,undefs` (or ANDROID_ALLOW_UNDEFINED_SYMBOLS) — check at
  implementation.
- Static dep .a objects must be PIC to link into plugins — the NDK
  compiles everything PIC by default; verify once (readelf) and only
  add CMAKE_POSITION_INDEPENDENT_CODE if actually needed.

## Gradle/app changes

- `externalNativeBuild targets`: replace `cmus_deps` with the cmus
  executable + the 10 plugin targets (statics come transitively;
  `cmus_deps` custom target retired or kept for convenience — decide).
- `packagingOptions { jniLibs { useLegacyPackaging = true } }` so
  everything is extracted to nativeLibraryDir as real files (exec +
  symlink targets need this; sets extractNativeLibs in the manifest).

## Verify

- Clean `./gradlew clean assembleDebug`; APK gains exactly
  `lib/arm64-v8a/libcmus.so` + 10 `libcmus_{ip,op}_*.so`; terminfo
  asset unchanged.
- readelf sanity: libcmus.so is ET_DYN/PIE with cmus symbols in
  .dynsym; a plugin shows undefined core syms + its codec statically
  linked (e.g. FLAC__stream_decoder_* defined inside cmus_ip_flac).
- On-device (adb, real device): build the stage-6 runtime layout by
  hand under /data/local/tmp — `{ip,op}/` dirs with plugin copies under
  their short names, data/ push of cmus `data/`, empty CMUS_HOME,
  terminfo from assets; env TERM/TERMINFO/CMUS_{HOME,LIB_DIR,DATA_DIR}.
  Run in an interactive adb shell: TUI renders, all plugins load with
  no dlopen errors (:set output_plugin shows aaudio), add + play a
  flac and an mp3, seek/pause. If shell-uid audio policy blocks aaudio
  output, record it and verify op init/error paths only — real
  playback then lands with stage 6 in-app.
- socat smoke-test against the server.c socket (pre-stage-7 baseline).

## Risks / decide at implementation

- AGP 9.3 might package only declared SHARED-target artifacts rather
  than globbing lib*.so from the output dir — fallback: POST_BUILD
  copy of the executable into a jniLibs srcDir, or a dummy SHARED
  wrapper. Resolve empirically first thing.
- aaudio playback permission from the adb shell uid.
- Whether faad/mp4v2 static objects need anything extra when linked
  into two different plugin .so files (duplicate static state is fine
  — each plugin is self-contained by design).
- cmus warning flags: adopt upstream's set (configure check_cflags)
  where clang accepts them; don't fight new-clang warnings in pristine
  sources — no patches this stage.
